## часть 1
 
### 1. Инициализация репозитория
```
git init
ls
ls -la
git status
```
##### Что было сделано:

* Выполнена инициализация нового Git-репозитория (git init).
* Проверено содержимое директории (ls, ls -la).
* Проверено текущее состояние репозитория (git status) — репозиторий пуст, отслеживаемых файлов нет.

---

### 2. Создание и добавление первого файла
```
touch readme.txt
nano readme.txt
git status
git add readme.txt
git status
git commit -m "First file"
git status
```
##### Что было сделано:

* Создан файл readme.txt.

* Файл отредактирован через nano.

* Проверен статус.

* Файл добавлен в индекс (git add).

* Создан первый коммит с сообщением "First file".

* Репозиторий перешёл в чистое состояние.

---

### 3. Изменение файла
```
nano readme.txt   # добавили "hello, world!"
git status
git add readme.txt
```
##### Что было сделано:

* В файл добавлена строка hello, world!

* Git зафиксировал изменения.

* Обновлённая версия файла добавлена в staging area.

---

### 4. Создание дополнительных файлов

В директории вручную созданы два файла:

* readme after.txt

* readme now.txt

Далее:
```
git status
git add '*.txt'
git status
git commit -m "All files with txt extension were added"
```
##### Что было сделано:

* Проверено состояние репозитория.
* Добавлены все файлы с расширением .txt с использованием wildcard *.txt.
* Создан коммит с сообщением: "All files with txt extension were added"

---

### 5. Просмотр истории коммитов

```
git log
git log --summary
```
##### Что было сделано:

* Просмотрена история коммитов.
* Выведена расширенная информация о внесённых изменениях.

---

### 6. Подключение удалённого репозитория и отправка изменений

```
git remote add origin <ссылка на репозиторий>
git push -u origin main  # в видео master 
```
##### Что было сделано:

* Добавлен удалённый репозиторий origin на платформе GitHub.
* Выполнена первая отправка изменений (push) в ветку master (или main).
* Использованы логин и персональный токен доступа.
* Проверено наличие файлов в веб-интерфейсе GitHub.

---

### 7. Изменение файла через GitHub и синхронизация
 
В браузере изменён файл readme.txt.
Теперь в нём 3 строки, каждая содержит:
```
hello, world!
```
Далее: 
```
git pull origin master
git diff HEAD
```
##### Что было сделано:

* Получены изменения из удалённого репозитория.
* Просмотрены отличия между текущим состоянием и последним коммитом.

---

### 8. Локальное изменение файла
 
```
nano readme.txt
```
Удалены восклицательные знаки.
Теперь файл содержит:
```
hello,world
hello,world
hello,world
```
Проверка изменений:
```
git diff HEAD
```

---

### 9. Работа с новой директорией
 
```
mkdir folder
cd folder
touch bye.txt
git status
git add folder/bye.txt
cd ..
git status
git diff --staged
```
##### Что было сделано:

* Создана папка folder.
* Внутри создан файл bye.txt.
* Файл добавлен в staging area.
* Проверены индексированные изменения.

---

### 10. Отмена изменений
 
```
git reset folder/bye.txt
git diff --staged
git diff
git checkout -- readme.txt
git status
cat readme.txt
```
##### Что было сделано:

* Отменено добавление bye.txt в индекс.
* Проверено состояние staged-изменений.
* Отменены изменения в readme.txt.
* Подтверждено восстановление файла к последнему коммиту.

---

### 11. Работа с ветками

```
git branch clean_up
git branch
git checkout clean_up
git branch
ls
```
##### Что было сделано:

* Создана новая ветка clean_up.
* Переключение на неё.
* Проверка активной ветки.

---

### 12. Удаление файлов и коммит

```
rm -r folder
git commit -m "Deleted folder and files"

git rm readme\ now.txt
git rm now\ now.txt
git status
git commit -m "Deleted folder and files"
```
##### Что было сделано:

* Удалена папка folder.
* Удалены файлы через git rm.
* Зафиксированы изменения отдельным коммитом.

---

### 13. Слияние веток

```
git checkout main   # в видео master
git merge clean_up
git branch -d clean_up
git push
```
##### Что было сделано:

* Переключение на основную ветку.
* Выполнено слияние ветки clean_up.
* Удалена временная ветка.
* Изменения отправлены в удалённый репозиторий.

---

### 14. Проверка результата

```
git checkout main   # в видео master
git merge clean_up
git branch -d clean_up
git push
```
##### После выполнения git push:

* Веб-интерфейс GitHub подтверждает:

  *  Удаление файлов
  *  Актуальное состояние репозитория
  *  Синхронизацию локальной и удалённой версий

### Функциональные возможности, задействованные в работе

* Управление версиями

  *  Отслеживание изменений файлов (diff viewer)
  *  Индексация изменений (staging area)
  *  Формирование логичной истории коммитов

* Работа с ветками

  *  Создание feature-веток
  *  Переключение между ветками
  *  Слияние (merge) веток

* Синхронизация с удаленным репозиторием

  *  Push локальных изменений
  *  Pull обновлений
  *  Fetch изменений без автоматического merge

### Преимущества использования

* Интуитивный графический интерфейс.
* Наглядность изменений.
* Повышение скорости рутинных операций — commit, push, pull выполняются в несколько кликов.
